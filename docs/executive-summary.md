# Executive Summary

This case study explains how a web development agency can use the NIST Cybersecurity Framework (CSF) 2.0 to manage cybersecurity risk in a practical and structured way.

The organization used in this report, H&L Development (HLD), is a model web development agency that designs and maintains enterprise web applications, authentication portals, cloud-hosted systems, and e-commerce integrations. Its business operations rely on critical digital assets such as customer login systems, cloud databases, source code repositories, and third-party API keys.

Using the NIST CSF, the organization can first **identify** its critical assets, dependencies, threats, vulnerabilities, and business risks. This step helps HLD understand what must be protected and where the greatest exposures exist. In this scenario, the most important risks include credential stuffing, data exfiltration, supply chain compromise, and leaked API secrets. Those risks can then be tracked in a register using inherent and residual risk, treatment status, ownership, and review cadence.

The next step is to **protect** these assets through access controls, encryption, secure development practices, secret management, branch protection, peer review, rate limiting, CAPTCHA, and multi-factor authentication. These controls reduce the chance that an attacker will successfully compromise the organization and help move higher-risk items toward a lower residual risk.

The organization must also be able to **detect and respond** to cybersecurity events. HLD improves visibility by using centralized logging, SIEM, anomaly detection, alerting, and defined incident response procedures. Once suspicious activity is detected, the organization classifies the incident, contains the threat, removes the root cause, communicates with internal and external stakeholders, and documents the event. That response work is what turns a written control set into an operational security process.

Finally, the organization must be able to **recover**. HLD restores systems from validated backups, prioritizes critical customer-facing services, communicates recovery status, and performs a lessons-learned review. Recovery is not only about getting systems back online. It is also about returning them to a secure state and improving resilience for the future. A mature recovery process also feeds updates back into the risk register and control plan.

Overall, the NIST CSF gives HLD a practical structure for turning broad cybersecurity goals into concrete decisions, controls, workflows, and improvement actions.
